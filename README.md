# Fluxum - Um novo desenvolvimento, uma nova empresa, um novo universo.
- Fluxum, uma startup focada em ailiar pequenas empresas que tem dificuldade em entrar no mercado digital. Nossa proposta é oferecer modernidade, agilidade e acima de tudo, facilidade. A empresa que participar do projeto, vai contar com suporte na parte de gestão de finanças, controle de atividades, informações do me, estatísticas de pagamentos e listagem de funcionários, tudo como forma de controle para as informações respectivas do usuário.

2. Briefing e Análise
  2.1 OBJETIVO GERAL: Fluxum, uma startup focada em ailiar pequenas empresas que tem dificuldade em entrar no mercado digital. Nossa proposta é oferecer modernidade, agilidade e acima de tudo, facilidade. A empresa que participar do projeto, vai contar com suporte na parte de gestão de finanças, controle de atividades, informações do me, estatísticas de pagamentos e listagem de funcionários, tudo como forma de controle para as informações respectivas do usuário.
  2.1.1 Problema: Pensamos em pequenas empresas que não tem um controle de gerenciamento de vendas ou uma mídia digital centrada. Empresas que, de início, precisam de um apoio digital.
  2.2.2 Solução: A fluxum oferece um plano completo de gerenciamento digital, desde um site já online, sistema de registro de vendas com conexões através de planos via whatsapp.

  2.2 PÚBLICO-ALVO: Empresas pequenas que ainda não tem uma marca digital e comunidades que não possuem aplicações ou buscam uma experiência mais imersiva e fácil.

    2.3.3 Para empresas de serviços que oferecem informações em tempo real (projeto 03): Sistemas IRP, IPR

3. MODELO DE MONETIZAÇÃO:

4. Requisitos do sistema
Requisitos Funcionais
RF01 - O sistema deve mostrar a página inicial com as informações da empresa. Primeira página vista pelo visitante, apresentando de forma resumida quem é a Fluxum.
RF02 - O sistema deve apresentar os serviços que a Fluxum oferece. Que lista os serviços, ajudando o visitante a entender a  empresa.
RF03 - O sistema deve mostrar os produtos e soluções disponíveis para os clientes. Área do site com os produtos/soluções da empresa, incluindo nome e descrição de cada um.
RF04 - O usuário deve conseguir navegar pelas páginas usando o menu. Menu presente em todas as páginas, com links para as principais seções do site.
RF05 - O sistema deve mostrar as informações de contato da empresa, como telefone, endereço e e-mail. Permite que o visitante encontre facilmente como falar com a empresa.
RF06 - O sistema deve permitir que o administrador realize o cadastro de empresas. O administrador preenche um formulário com os dados da empresa e o sistema salva as informações.
RF07 - O sistema deve permitir que o administrador realize o cadastro de produtos. Semelhante ao cadastro de empresas, mas voltado a produtos (nome, descrição, categoria, imagem).
RF08 - O sistema deve verificar se os campos obrigatórios foram preenchidos antes de enviar o cadastro, impedindo o envio caso algum esteja vazio. Confere se os campos essenciais foram preenchidos antes de salvar o cadastro.
RF09 - O sistema deve mostrar uma mensagem informando que o cadastro foi realizado com sucesso. Confirma ao administrador que o cadastro foi salvo corretamente.
RF10 - O usuário deve conseguir visualizar os produtos cadastrados. Produtos cadastrados ficam visíveis na área pública do site, formando o catálogo.
RF11 - O botão "Saiba mais” deve levar o usuário para a seção com mais informações sobre a startup. Direciona o visitante até uma seção com detalhes sobre a história e os valores da Fluxum.
RF12 – O uso de Busca e filtros de produtos. O sistema deve permitir que o administrador faça login para acessar a área de cadastros.

Requisitos Não Funcionais
RNF01 - a interface deve ser simples, organizada e intuitiva, permitindo que um usuário sem treinamento prévio consiga navegar sem dificuldades. Ninguém deve precisar de instruções para usar o site — a navegação tem que ser óbvia por si só.
RF02 - textos, botões e elementos de interação devem ter tamanho e contraste adequados para boa legibilidade. Garante que qualquer pessoa, inclusive com dificuldade visual leve, consiga ler e clicar sem esforço.
RNF03 - o site deve manter o mesmo padrão visual (cores, tipografia e layout) em todas as páginas. Evita que o site pareça "quebrado" ou inconsistente ao trocar de página, reforçando a identidade visual da marca.
RNF04 – o sistema deve ser responsivo, funcionando corretamente em desktop, tablet e celular. O layout se adapta ao tamanho da tela automaticamente, sem cortar conteúdo ou quebrar o design.
RNF05 - o sistema deve funcionar corretamente nas versões atuais dos navegadores chrome. Garante que a maioria dos visitantes, independente do navegador escolhido, tenha a mesma experiência.
RNF06 - as páginas devem carregar em até 3 segundos em uma conexão de banda larga padrão. Tempo de carregamento longo aumenta a chance do visitante desistir e sair do site.
RNF07 – o sistema deve suportar pelo menos 100 usuários acessando simultaneamente sem perda perceptível de desempenho. Evita lentidão ou travamentos em momentos de maior acesso ao site.
RNF08 -  as senhas de acesso à área administrativa devem ser armazenadas de forma criptografada. Mesmo que o banco de dados seja comprometido, as senhas não ficam expostas em texto legível.
RNF09 - os dados cadastrados devem ser armazenados em banco de dados com controle de acesso restrito à área administrativa. Só o administrador autenticado pode alterar os cadastros, que protege as informações de acessos indevidos.
RNF10 – o sistema deve exibir mensagens de erro claras quando um campo for preenchido incorretamente. Ajuda o usuário a entender e corrigir o problema no formulário sem ficar confuso.
RNF11 -  o código deve seguir um padrão de organização (Ex: separação por módulos/camadas) que facilite manutenção e novas funcionalidades. Facilita que outros desenvolvedores entendam e modifiquem o sistema no futuro, sem retrabalho.
RNF12 - O sistema deve manter disponibilidade mínima de 99% do tempo, excluindo manutenções programadas. Garante que o site esteja praticamente sempre no ar, com paradas raras e planejadas.
